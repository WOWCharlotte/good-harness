# 职责设计原则

综合所有文章和项目，可以形成一组 Agent 职责设计原则。

第一，目标先于行动。Agent 不应在目标不清时直接动手。

第二，调查先于修改。尤其是工程任务，读代码和理解上下文比生成补丁更重要。

第三，工具调用必须结构化。自然语言工具描述不能替代 schema、校验和权限。

第四，权限边界不能交给模型自觉。高风险动作必须由 Harness 管理。

第五，记忆是提示，不是事实。长期记忆需要验证、分类和审计。

第六，完成必须由验证确认。Agent 的完成声明必须带证据。

第七，多 Agent 必须有协议。没有任务图、通信格式和结果压缩，多 Agent 只会放大混乱。

第八，子 Agent 要隔离上下文和工具。隔离是多 Agent 的核心安全机制。

第九，失败要记录为数据。失败不是丢脸，而是改进 Harness 的输入。

第十，人类必须保留追问目标的权力。Agent 可以执行目标，但不应垄断目标是否合理的判断。

这些原则共同指向一个结论：Agent 的职责不是越多越好，而是越清晰越好。职责清晰，Agent 才能在边界内发挥能力；职责模糊，自主性就会变成风险。
