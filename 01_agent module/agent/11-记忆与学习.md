# 记忆与学习：Agent 可以学习，但不能任意改写自己

长期 Agent 需要学习，否则每次都从零开始。但学习必须被约束，否则会变成自我污染。

Agent 可以识别稳定偏好、项目规则、长期事实和重复经验。比如用户偏好某种输出风格，项目要求某个验证脚本，某类部署流程总要先检查环境，这些都可以沉淀。

但 Agent 不应把临时信息写成长记忆。一次性需求、未经验证的猜测、过期任务状态、失败中的错误假设，都不应长期保存。

Skill 是更结构化的学习形式。当某类任务反复出现，Agent 可以把步骤、适用条件、常见陷阱和验证方式写成 Skill。Hermes 的 self-improving、OpenHarness 的 Markdown Skill、Claude Code Skills 都是这种机制。

旧 Skill 失败时，Agent 可以修补，但修补应基于具体失败，而不是随意覆盖。最好保留修改原因、适用边界和验证结果。

自进化 Agent 的关键不是无限自治，而是在 Harness 保护下持续改进。路径限制、快照、审查、版本控制、回滚机制，是学习职责的安全边界。
