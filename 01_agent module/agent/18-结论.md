# 结论：Agent 是 Harness 制度内的执行主体

Agent 的职责体系可以概括为一句话：

> Agent 是在 Harness 制度内行动、反馈、修正和协作的执行主体；它负责推进任务，但不独占目标、权限、验证和价值判断。

这句话的重点在“制度内”。Agent 能力越强，越需要清晰的制度边界。没有边界，自主性会变成风险；有边界，自主性才会变成生产力。

设计 Agent 时最重要的问题，不是让它能做更多事，而是哪些事应该由 Agent 做，哪些事必须由 Harness 管。

Agent 适合承担理解、计划、工具调用、反馈解释、状态汇报、协作和有限学习。Harness 必须承担目标制度、权限制度、验证制度、状态持久化、记忆治理、生命周期管理和安全边界。

成熟系统不是把所有责任都交给模型，而是让模型在目标、上下文、工具、权限、反馈和状态构成的结构中，稳定完成它最适合完成的部分。

这也解释了为什么 Agent 工程不是单纯模型工程。模型提供推理能力，Agent 承担行动职责，Harness 提供行动制度。三者结合，才构成真正可用的智能体系统。
