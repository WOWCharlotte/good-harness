# Agent 不应承担的职责

要设计清楚 Agent 职责，也必须说清楚它不应承担什么。

Agent 不应独自决定目标函数。目标必须来自用户、组织、任务制度或评测体系。否则 Agent 会优化它自己理解的目标，而不是系统真正需要的目标。

Agent 不应绕过权限系统。权限是 Harness 的边界，不是模型的道德自觉。高风险动作必须由系统批准、沙箱或网关控制。

Agent 不应隐藏失败。失败是反馈闭环的数据。如果 Agent 为了“完成感”掩盖失败，系统就无法改进。

Agent 不应把未经验证的记忆当事实。记忆只能作为提示，关键判断仍需回到代码、测试、日志或用户确认。

Agent 不应在不透明后台长期行动。后台任务必须有日志、状态、通知和停止机制。

Agent 不应无限递归委派。多 Agent 系统需要深度限制、工具限制和结果协议。

Agent 不应把“文本看起来合理”当完成。完成必须回到验证信号。

这些“不应承担”的职责，实际上定义了 Harness 的责任。Agent 越强，越需要清晰地知道哪些事不能由它自己决定。
